# Construction-Turtle Roadmap — Q2-Q3 2026

> One-line summary: an 8-week sequenced build plan that takes Chad's app from "morning brief + chat + dashboard" to "AI-native, no-portal-for-anyone field workflow nobody else has shipped." Derived from the May 2026 competitor research.

**Status:** Active build plan.
**Created:** 2026-05-11.
**Owner:** CP.
**Anchors:**
- ADR-POS-001 — Construction-Turtle Positioning (in `~/Projects/patton-os/data/decisions.md`) — locks the AI-native, no-portal, single-builder, iPad-native shape.
- [competitor-research-2026-05-11.md](competitor-research-2026-05-11.md) — Buildertrend + 7 PM tools.
- [competitor-research-photos-2026-05-11.md](competitor-research-photos-2026-05-11.md) — CompanyCam + 6 photo tools.

## TL;DR

Two independent market scans concluded the incumbent residential construction software is CRUD web apps with bolted-on AI. **The wedge is product shape, not feature surface.** This roadmap is the eight features both reports independently ranked as highest-leverage, sequenced so each week ships something demoable and the strategic moats (SMS sub coordination, AI-native walkthrough, iPad-native) accumulate.

| Week | Theme | Headline deliverable |
|---|---|---|
| 1 | GPS auto-bind + photo→site-log | "Pull into Whitfield → app already knows" |
| 2 | SMS sub coordination MVP | Subs text photos; lands in Chad's project tagged with sender, zero login |
| 3 | Selections module (lean) | Selections list per project with drop-dead alerts on the existing trigger engine |
| 4 | Voice walkthrough → AI PDF | Chad does 5-min walkthrough on iPad; gets a styled PDF in inbox |
| 5 | Weekly client snapshot email | Friday morning, homeowners get a styled status update — no portal |
| 6 | Before/after photo capture | Ghost-overlay capture + slide-to-reveal UX |
| 7 | iPad split-view + Pencil | iPad-native project nav + photo annotation |
| 8 | AI photo captions + auto-tag | Every photo gets a Claude-vision-generated description and tags |

Eight weeks. Eight visibly-different demos. Three of them (Week 2, Week 4, Week 7) are genuinely uncopyable by incumbents inside a 12-month window because they require a re-platform, not a feature flag.

## Week-by-week

### Week 1 — GPS-geofence auto-bind + photo→site-log auto-attach

**Why first:** Both research reports flagged "which project?" friction as the #1 demo killer. Two small features that build foundation for everything else.

**Scope:**
- iOS: `LocationManager` + `CurrentProjectStore` ([already on `feat/gps-geofence` branch](specs/gps-geofence-auto-bind.md))
- Backend: `010_project_geofences.sql` migration adding `lat`, `lng`, `geofence_radius_m` to `projects`. New `GET /v1/turtles/{tid}/projects/with-locations` endpoint.
- Photo capture flow: consume `CurrentProjectStore.currentProjectID` to pre-select.
- Site-log auto-attach: today's photos taken inside an active geofence land as the day's site log attachments without manual upload.

**Effort:** S (iOS scaffolding already exists from this week's research session).

**Success criteria:**
1. Schema migration ships clean.
2. Crossing Whitfield's geofence sets `currentProjectID` within ~30s.
3. Open Camera tab → Whitfield is pre-selected, no picker tap.
4. Photo taken during the day attaches to today's Whitfield site log automatically.
5. Manual override still works; geofence is a hint, not a lock.

**Dependencies:** None.

**Deferred to v2:** Background launch + push notification on enter ("You're at Whitfield — log a note?"). Tap-on-map geocoding in iOS. Polygon geofences for irregular lots.

### Week 2 — SMS sub coordination MVP

**Why second:** The competitor research called this the largest unfilled gap in the entire market. **Every** review of every tool says some version of "the subs won't log in to the portal." Twilio is well-understood plumbing — this is a market gap a single developer can ship in a week, that even Buildertrend can't ship in a year (their entire architecture assumes user accounts).

**Scope:**
- Provision a Twilio phone number (one number per turtle for now).
- FastAPI webhook `POST /v1/sms/inbound` parses MMS + body, identifies sender by phone, attempts project bind via geofence-of-cell-tower, falls back to "ask Chad to confirm" message.
- Per-project magic-link URLs: `https://chad.pattonai.app/sub/{project_id}/{token}` — opens a stateless web form that subs can use without an account (token expires in 24h).
- Photo MMS → `photos` table tagged with project + sender. Lands as a candidate for that day's site log.
- Sub identity store: phone number → known sub name. Auto-populated from emails Chad sends to subs (the inbox watcher already sees these).
- iOS: new "Pending SMS uploads" badge in the inbox view; tap to approve / reject / re-tag.

**Effort:** M (1 week, mostly server-side). 

**Success criteria:**
1. Chad shares his Twilio number with a real sub on a real job.
2. Sub texts a photo. Within 5 minutes, the photo appears in Chad's app tagged with sender + project (or marked as needing Chad's confirmation if auto-bind failed).
3. Sub gets an auto-reply with the magic-link URL for the project's drop-folder.
4. No account, no login, no app install — ever.

**Dependencies:** Week 1's `currentProjectID` / `CurrentProjectStore` is reused server-side for the cell-tower → project heuristic.

**Demo value:** This is the feature you show to RPM Filter, future commercial-HVAC prospects, every future builder client. It's the one slide that ends the meeting.

### Week 3 — Selections module (lean)

**Why third:** Every custom-builder review of Buildertrend praises the Selections module. Chad's biggest current gap vs incumbents. The existing drop-dead trigger engine does 80% of the data work already — we just need a thin CRUD layer + view.

**Scope:**
- New Postgres table `selections` with columns: `project_id`, `room`, `item`, `vendor`, `cost_estimate`, `selection_deadline`, `status` (pending|confirmed|installed), `notes`.
- Wire `selections.selection_deadline` into the existing `fire_selection_deadlines_for_project()` engine — same drop-dead alerts as material order dates.
- CLI: `hb-selections add / list / set-status / set-deadline`.
- iOS: new "Selections" tab inside ProjectDetailView. Per-room grouping. Status chips. Tap a deadline → see the days-until-overdue calculation.
- The Help Desk agent ingests "what's pending in master bath?" → answers from selections table.

**Effort:** M.

**Success criteria:**
1. Chad can add a selection in <20s via voice ("master bath tile, Lowe's, $1200, due Aug 15").
2. Drop-dead alert fires per the existing trigger engine without code changes to that engine.
3. iOS Selections view renders in <300ms for a project with 50+ selections.
4. Help Desk agent answers selection questions from the structured table, not from RAG.

**Dependencies:** None.

### Week 4 — Voice walkthrough → AI-generated PDF report

**Why fourth:** CompanyCam Sidekick's flagship feature (Premium tier, $129/mo). Voice-narrated walkthrough → polished PDF for homeowners. Chad's app already has voice mode + photo capture + Claude — this is wiring them together, not building new primitives.

**Scope:**
- iOS: new "Walkthrough" action in Camera tab. Records voice + captures photos with timestamps + GPS.
- Backend: new `walkthrough_agent.py`. Takes the audio (Whisper transcription) + photos + project context → produces a Sidekick-style PDF using the existing `branded_pdf` template.
- PDF structure: title, date, summary paragraph, per-photo section with caption + Chad's voice note transcribed, signature footer.
- Delivery: PDF emails to Chad + optionally to a homeowner (Chad approves before send).
- Cost target: <$0.50 per walkthrough (Whisper + Claude Sonnet for synth).

**Effort:** M.

**Success criteria:**
1. Chad does a 5-min walkthrough on iPhone, gets a PDF in his inbox within 90 seconds.
2. PDF reads like Chad wrote it (voice rubric stays consistent with morning brief and homeowner emails).
3. Photo placement matches voice narration order.
4. Cost per walkthrough stays under $0.50 at typical 8-12 photo runs.

**Dependencies:** Week 1 (GPS auto-bind for project tag), Week 8 photo-caption agent (can land independently — Week 4 uses voice narration; Week 8 just supplements).

### Week 5 — Weekly client snapshot email

**Why fifth:** Mirrors Buildertrend's "AI client update" without building the portal nobody uses. The morning-brief agent already synthesizes for Chad — this is a sibling agent that synthesizes for homeowners. Closes the "client communication" gap with no auth surface.

**Scope:**
- New `client_brief_agent.py`. Runs Fridays at 7 AM. Per active project, pulls last 7 days of: photos taken on-site, site log entries, change orders drafted, drop-dead alerts triggered, cost-tracker deltas.
- Synthesizes a homeowner-friendly status email: progress paragraph, this week's photos as inline thumbnails, next week's preview, open questions Chad has for them.
- Email-only delivery. NO portal. NO links to a web app. Inline images, plain text-friendly fallback.
- Chad approves before send (via the existing approval-poll mechanism — same as CINO).
- Per-project opt-in/opt-out flag.

**Effort:** S.

**Success criteria:**
1. Friday 7 AM: drafts appear in Chad's pending-approvals queue.
2. Chad approves → email lands in homeowner's inbox.
3. Homeowner can reply directly — the inbox watcher already drafts Chad's response.
4. No homeowner ever needs to log into anything.

**Dependencies:** None directly. Benefits from Week 4 walkthrough PDFs as an attachment option.

### Week 6 — Before/after photo capture (ghost overlay + slide-to-reveal)

**Why sixth:** Trivial to build, huge marketing payoff for a luxury custom-homes client. The before/after slider is THE social-media construction shot. Nobody else automates the alignment.

**Scope:**
- iOS: extend Camera tab. When user opens camera at a location with a previous photo (via reverse-geocoded coordinates + room tag), show the previous photo as a 30% opacity ghost overlay so the user aligns the new photo to the same vantage.
- New view: "Before/After" gallery. Slide-to-reveal UX (touch-and-drag a vertical divider).
- Export: tap a before/after pair → generates a side-by-side or animated GIF for Chad's marketing.
- Auto-grouping: photos from the same room across time become before/after candidates.

**Effort:** S.

**Success criteria:**
1. Open camera in Whitfield's master bath → ghost overlay of last master bath photo appears.
2. Tap photo timeline → swipe through before/after pairs.
3. Export a slide-to-reveal as a shareable URL or GIF in <5s.

**Dependencies:** Week 1 (geofence auto-bind), Week 4 (photo metadata already includes room tags).

### Week 7 — iPad split-view + Pencil annotation

**Why seventh:** Every research report flagged iPad-native as universally weak. Luxury custom homes is the customer segment where iPad is actually used in the field (the plans review, the homeowner meeting). Pencil annotation on photos turns the iPad into a sales tool.

**Scope:**
- iOS: detect iPad form factor in app launch. Switch root view to a `NavigationSplitView` (sidebar: project list; primary: project detail; secondary: photo/document viewer).
- Photo viewer: tap "Annotate" → enter Pencil mode. PencilKit canvas overlaid on photo. Save → photo + annotation layer persisted as a combined image.
- Pencil-specific: hover preview, double-tap for eraser, pressure sensitivity.
- Plans review: when Chad opens a PDF plan, Pencil annotations save to the project's Drive folder automatically.

**Effort:** M.

**Success criteria:**
1. Launch app on iPad → split-view layout. Launch on iPhone → no regression.
2. Pencil annotation on a photo persists across app launches and syncs to Drive.
3. Annotations appear in the photo's metadata so the AI walkthrough agent can reference them ("Chad circled the cabinet pull location on the kitchen photo").

**Dependencies:** None.

---

### Week 8 — AI photo captions + auto-tag

**Why eighth:** Closes parity with CompanyCam Sidekick. Claude vision is cheap (<$0.01/photo). The captions feed Week 4's walkthrough PDFs, Week 5's homeowner emails, and the photo search experience.

**Scope:**
- New `photo_caption_agent.py`. Runs async on every photo upload. Sends to Claude Sonnet vision with project context → returns: 1-sentence caption + 3-5 structured tags (phase, room, subject).
- Caption shown inline in photo grid + searchable via existing chat ("show me framing photos with rebar").
- Tags drive auto-grouping for before/after (Week 6) and walkthrough PDFs (Week 4).
- Confidence threshold: low-confidence captions surface as "AI suggested — confirm" rather than auto-saved.

**Effort:** M.

**Success criteria:**
1. Take a photo → within 10s, a caption + tags appear under the photo.
2. Chat query "framing photos with cabinet preview" returns the right photos.
3. Caption accuracy ≥80% on a 50-photo eval set Chad scores after the first week.

**Dependencies:** None directly.

## What's NOT in this plan (binding via ADR-POS-001)

These are NOT on the roadmap and won't be added without an explicit ADR override:

| Skipped feature | Why |
|---|---|
| Gantt chart / scheduling grid | Buildertrend has it. It's not our wedge. We don't compete on parity. |
| Formal RFI module | Incumbents have it. Chad's email + inbox watcher already covers the workflow informally. |
| Sub portal with login | Portal-fatigue is universally cited. SMS (Week 2) is the wedge. |
| Homeowner web portal | Email + the walkthrough PDFs (Week 4) + weekly snapshot (Week 5) cover this without auth surface. |
| Lead pipeline / CRM | Not a builder problem on our scale. Chad doesn't need leads — he needs to ship the ones he has. |
| Time clock / payroll | Different software category entirely. Integrations are fine; building it is scope creep. |
| LiDAR scanning | Niche, expensive, not a Palmetto problem. |
| Plans overlay / floor-plan markup | Pencil on plan PDFs (Week 7) is the leaner version. Floor-plan markup is full-product territory. |
| Marketing / lead-gen suite | Houzz Pro's territory. Different buyer. |

## After Week 8 — backlog candidates

In rough priority order, but not committed:

1. **Punch list module** — site walks generate auto-detected punch items via AI vision on photos. ("This door reveal is uneven" — Claude vision can flag these.)
2. **Inspection coordination** — local-permit-office integrations, inspection date alerts, inspector contact directory.
3. **Background launch on geofence enter** — push notification ("You're at Whitfield. Open Site Log?") + foreground app pre-load.
4. **Cell-tower SMS location for subs** — Twilio Add-on for carrier-provided cell location on inbound MMS so we improve sub project-binding from ~70% to ~95%.
5. **Voice-driven change orders** — extend the existing change-order agent with voice input.
6. **Walkthrough PDF homeowner approval flow** — homeowner replies "approved" to the walkthrough email → marked as accepted in change-order state.
7. **Inspector / homeowner SMS too** — extend Week 2 SMS to non-sub roles.
8. **Multi-tenant Patton AI Platform consolidation** — patton-os + home-builder-agent + future turtles all share auth, telemetry, and platform-level features (per IP-001).

## Pace expectations

- **One full week of focused build per roadmap week.** Connor is single-developer; this is realistic only if no client crises divert him.
- **Each week ends with a demoable build on TestFlight.** No dark-launched features. The demo IS the proof.
- **Costs:** Per-feature LLM cost is bounded by the prompt caching we shipped 2026-05-11. Total monthly LLM spend at 2x activity should stay <$50/month even after all 8 features ship.
- **Reviews:** After Week 4 (mid-roadmap) and Week 8 (end), spawn a competitor-research agent again to see what shifted in the market. The May 2026 snapshot will be stale within 90 days.
